## 2. TECHNOLOGY STACK & TOOLING IMPLEMENTATION

### A. Frontend: React.js + Vite + TailwindCSS
- **Purpose**: To provide a highly responsive, Single Page Application (SPA) dashboard.
- **Implementation**: 
  - `Vite` was chosen over Webpack/CRA because of its native ES-Module hot-module replacement (HMR), drastically speeding up development times.
  - State management is handled natively with React `useState` hooks since the data flow is strictly Top-Down (Form -> Parent App Component -> Result Tabs).
  - Designed using `TailwindCSS` for rapid UI prototyping, utilizing utility classes for dark-mode glassmorphism and flexbox grids.
- **Key Libraries**: 
  - `axios`: For making asynchronous Promise-based HTTP requests to the backend.
  - `mermaid.js`: A JavaScript-based diagramming and charting tool. It processes the text-based Graph TD string returned by the AI and renders the SVG architecture flow natively in the DOM.
  - `react-syntax-highlighter`: To display the generated Terraform code with standard IDE-like syntax highlighting.

### B. Backend: Python + FastAPI
- **Purpose**: Handles incoming application constraints, manages third-party I/O (OpenAI), and synthesizes code templates.
- **Implementation**:
  - `FastAPI` was chosen due to its asynchronous runtime (ASGI) and native Pydantic integration, making JSON validation effortless and fast.
  - Exposes a `POST /generate` endpoint accepting a strictly typed Pydantic Class (`ArchitectureRequest`).
- **Key Services**:
  - **AI Generator (`openai`)**: Interacts with the GPT models. It passes a heavily engineered prompt specifying that the output must *strictly* be a verifiable JSON schema. 
  - **Terraform Generator (`Jinja2`)**: Generates Infrastructure as Code (IaC). Instead of using string concatenation, it uses static `.tf.j2` templates containing variables (like `{{ compute_count }}`). The backend executes Jinja's rendering engine to inject parameters into the HashiCorp Configuration Language (HCL).
  - **Cost Estimator**: Currently a deterministic Python mapping. It iterates over the AI's returned JSON (`components` array) and multiplies counts by a static rate card dictionary depending on the cloud provider.

## 3. DIFFICULTIES & OBSTACLES FACED DURING DEVELOPMENT

### Obstacle 1: Unstable Environment and ESBuild Crashes (Frontend)
- **The Issue**: During the frontend initialization, dynamic dependencies like `mermaid`, `dompurify`, and `lodash-es` caused Vite's ESBuild pre-bundler to crash with a module resolution error (`Failed to resolve entry for package "dompurify"`). This happens because Mermaid internally requires heavy libraries that Vite attempts to resolve asynchronously on the fly.
- **The Solution**: 
  1. I had to manually clear the corrupted `node_modules` cache.
  2. I explicitly updated `vite.config.js` with the `optimizeDeps.include` block to force Vite to pre-bundle `mermaid`, `dompurify`, and `react-syntax-highlighter` synchronously before starting the dev server.

### Obstacle 2: OpenAI API Dependency & Startup Failure (Backend)
- **The Issue**: The `AsyncOpenAI` python client inherently assumes that an API key is available in the environment variables upon instantiation. Because the `.env` was empty initially, the FastAPI server threw an `OpenAIError` at compile-time and continuously shut down on port 8000.
- **The Solution**: I implemented a conditional instantiation fallback mechanism. The Python code now safely checks if `os.environ.get("OPENAI_API_KEY")` exists first. If it does not, it sets the client to `None` and catches the API calls to return a hardcoded "Mock Architecture" dict. This ensured the backend could gracefully boot up and serve the frontend even without live credentials.

### Obstacle 3: Constraining LLM Outputs to Valid JSON
- **The Issue**: Large Language Models (LLMs) are notorious for wrapping responses in conversational text or markdown blocks (e.g., \`\`\`json ... \`\`\`), which immediately breaks Python's `json.loads()`.
- **The Solution**: Formulated a rigid system prompt explicitly forbidding markdown wrapping. As a secondary safeguard, implemented a string sanitization block in Python that detects and strips leading conversational text or markdown ticks before parsing the JSON string.
